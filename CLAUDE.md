# 한국의 증류소 (Korean Distillery)

위스키 증류소 경영 시뮬레이션 게임. Godot 4.3+, GDScript, 2D 픽셀아트 (16x16 타일).

## 스토리

위스키를 좋아하는 평범한 개발자. AI에 밀려 퇴직당한다.
퇴직금을 들고 고민 끝에 작은 바(Bar)를 연다.

### 1막 — 바 운영 (꿈의 시작)

퇴직금으로 작은 바를 오픈. 위스키를 사입해서 손님에게 서빙하며 생계를 이어간다.
손님의 취향을 파악하고, 좋은 위스키를 추천하고, 단골을 만들어간다.
장사가 조금씩 잘 되기 시작하면서 더 좋은 위스키를 들이고 싶어진다.

**갈등**: 진짜 맛있는 위스키들은 물량이 없다. 유명 증류소는 소규모 바에 납품을 안 한다.
좋은 걸 팔고 싶은데 구할 수가 없다. 손님들은 점점 더 좋은 걸 원한다.

**전환점**: "까짓것, 내가 직접 만들자."

### 2막 — 증류소 (나만의 위스키)

바를 운영하면서 증류소를 시작한다. 처음엔 허름한 창고에 Pot Still 하나.
보리를 구하고, 증류하고, 오크통에 숙성하고... 첫 위스키가 완성되기까지의 여정.

바에서 번 돈으로 증류소에 투자. 증류소에서 만든 위스키를 바에서 판매.
두 사업이 서로를 먹여살리는 선순환 구조가 핵심.

**갈등**: 숙성에는 시간이 걸린다. 3년을 기다려야 첫 위스키가 나온다.
그동안 바 수입으로 버텨야 한다. 자금, 시간, 품질의 삼중고.

**전환점**: 첫 자체 위스키 출시. 손님들의 반응. "이거... 맛있는데?"

### 3막 — 대회 (증명)

자체 위스키가 입소문을 타기 시작한다. 평론가가 방문하고, 언론에 소개된다.
한국 위스키 품평회 초청장이 도착한다.

유명 증류소의 마스터 디스틸러들과 맞붙는다.
AI에 밀려났던 개발자가, 자기 손으로 만든 위스키로 인정받는 순간.

**클라이맥스**: 품평회 결선. 내 위스키 vs 전설의 위스키.

---

## 게임 컨셉

한국의 지역에서 바를 운영하고, 증류소를 세우고, 위스키를 만들어 최고의 증류소로 성장시키는 경영 시뮬레이션.

- 레퍼런스: 데이브 더 다이버 (낮-밤 이중 루프) + 스타듀밸리 (농사, 계절, 마을) + 흑백요리사 (대회)
- 시점: 탑다운 2D
- 아트: 픽셀아트 16x16 타일

## 게임 플로우 (3막 구조)

### 1막: 바 운영 (게임 시작 ~ Day 30 전후)

플레이어가 배우는 것:
- 위스키 종류와 풍미 (스모키, 스위트, 플로럴 등)
- 손님 취향 파악 & 추천
- 돈 관리 (사입 비용 vs 판매 수익)
- 바 업그레이드

하루 사이클 (1막):
```
오전: 시장에서 위스키 사입 (도매상 방문, 가격 비교)
오후: 바 준비 (재고 정리, 메뉴 구성)
저녁: 바 운영 (손님 서빙, 추천, 칵테일)
밤: 정산 (매출, 비용, 이익)
```

메카닉:
- 위스키 사입 시스템 (도매상에서 구매, 종류/가격/재고 한정)
- 손님 서빙 (취향 매칭, 추천, 만족도)
- 칵테일 제조 (레시피, 미니게임)
- 바 인테리어 & 분위기
- 단골 시스템 (자주 오는 손님 → 안정 수입)
- 평판 (리뷰, 별점, 입소문)

1막 진행 이벤트:
- Day 5~7: 첫 단골 생김
- Day 10~15: 손님이 특정 위스키 요청 → 도매상에 재고 없음 (복선)
- Day 15~20: 점점 더 좋은 위스키 수요 증가 → 구할 수 없는 것들 등장
- Day 25~30: 도매상 "그 위스키는 소규모 바에는 안 팔아요" → 전환점 이벤트
- 선택지: "증류소를 시작한다" → 2막 시작

### 2막: 증류소 시작 (Day 30~ Day 100 전후)

새로 배우는 것:
- 위스키 제조 공정 (몰팅 → 매싱 → 발효 → 증류 → 숙성)
- 원재료 수급 (보리 재배/구매, 이탄, 오크통)
- 숙성과 시간의 가치
- 바 수입 + 증류소 투자의 자금 관리

하루 사이클 (2막):
```
오전: 재료 수급 (보리밭, 시장, 채집)
오후: 증류소 작업 (제조 공정)
저녁: 바 운영 (1막과 동일, 자체 위스키 추가)
밤: 정산 + 숙성 경과 (하루 = 숙성 1개월)
```

메카닉 (1막 메카닉 + 추가):
- 위스키 제조 파이프라인 전체
- 보리밭/채집/시장 (원재료 수급)
- 오크통 관리 (종류 선택, 숙성 모니터링)
- 증류소 업그레이드 (Pot Still 추가, 창고 확장)
- 직원 고용 (바텐더, 증류 보조)
- 지역 특성 시스템 (기후 → 숙성 영향)

2막 진행 이벤트:
- 첫 증류 완료 → 오크통에 넣는 순간 (감동)
- 숙성 기다리는 동안 바 수입으로 버티기 (긴장감)
- 첫 자체 위스키 병입 → 바에서 첫 판매 (전환점)
- 단골 손님의 반응 이벤트
- 위스키 평론가 첫 방문

### 3막: 대회 & 성장 (Day 100~)

새로 배우는 것:
- 블렌딩 (여러 배럴 조합으로 시그니처 위스키)
- 경쟁과 전략 (라이벌 분석, 심사 기준 맞추기)
- 브랜드 빌딩

메카닉 (1+2막 메카닉 + 추가):
- 위스키 품평회 시스템 (예선/본선/결선)
- 라이벌 마스터 디스틸러 3인
- 블렌딩 시스템 (배럴 조합)
- 브랜드/명성 시스템
- 전국 유통 (대회 성적에 따라 해금)

라이벌 마스터 디스틸러:
- 전통파 '오 마스터': 스코틀랜드 수학. "한국에서 위스키라니..."
- 과학파 '닥터 캐스크': 화학 박사. 데이터 기반 증류
- 자연파 '와일드 그레인': 유기농, 자연 발효 철학

품평회 심사: 향(Nose) 30% + 맛(Palate) 30% + 피니시(Finish) 20% + 독창성 20%

## 위스키 제조 파이프라인 (2막에서 해금)

보리 → 몰팅(싹 틔우기) → 매싱(당분 추출) → 발효(효모) → 증류(Pot Still) → 숙성(오크통) → 블렌딩(3막) → 병입/판매

각 단계에서 플레이어의 선택이 최종 풍미 프로필에 영향:
- 이탄 사용량 → 스모키향 (0~100%)
- 효모 종류 → 과일향/꽃향
- 증류 컷 포인트 → 순도/캐릭터 (미니게임)
- 오크통 종류 → 바닐라/캐러멜/셰리
- 숙성 연수 → 깊이/복합미/가치

풍미 프로필 시스템:
```
whisky.flavor_profile = {
    "smoky": 0~100,     # 스모키/피티
    "sweet": 0~100,     # 달콤함 (바닐라, 캐러멜)
    "fruity": 0~100,    # 과일향
    "floral": 0~100,    # 꽃향
    "spicy": 0~100,     # 스파이시
    "marine": 0~100,    # 해안/짠맛 (부산 특화)
    "grain": 0~100,     # 곡물향 (전라 특화)
    "botanical": 0~100  # 보태니컬 (제주 특화)
}
```

## 성장 시스템 (Progression)

게임의 핵심 동기. 모든 시스템에 "돈 → 투자 → 업그레이드 → 더 많은 돈" 루프가 있어야 한다.
스타듀밸리처럼 시각적으로 공간이 변하고, 데이브 더 다이버처럼 메뉴/장비가 점점 좋아지는 느낌.

### 바(Bar) 업그레이드 트리 (1막~)

**공간 확장:**
```
Lv1: 허름한 바 — 좌석 4개, 메뉴판 칠판
Lv2: 아늑한 바 — 좌석 8개, 조명 추가, 간판
Lv3: 인기 바 — 좌석 12개, 인테리어 개선, 음악
Lv4: 프리미엄 바 — 좌석 16개, 프라이빗 룸, VIP석
Lv5: 전설의 바 — 좌석 20개+, 루프탑/테라스, 라이브 공연
```
업그레이드할 때마다 바의 타일맵이 실제로 변함 (시각적 성장 중요!)

**장비 업그레이드:**
```
잔(Glass):      기본 텀블러 → 글렌캐런 → 크리스탈 글라스     — 서빙 만족도 보너스
제빙기:         없음 → 기본 → 큰 얼음 → 구형 얼음 제조기     — 칵테일/하이볼 품질↑
탄산수 제조기:  없음 → 기본 → 프리미엄                       — 하이볼 매출 해금
칵테일 장비:    없음 → 기본 셰이커 → 전문 바 세트            — 칵테일 종류 해금
냉장고:         소형 → 중형 → 대형 → 와인셀러 겸용           — 재고 보관량↑
위스키 셀러:    없음 → 소형 진열장 → 대형 셀러               — 고가 위스키 보관 가능
인테리어:       없음 → 오크 카운터 → 무드등 → 벽난로          — 분위기 점수↑ → 팁↑
음향:           없음 → 스피커 → 턴테이블 → 라이브 스테이지    — 손님 체류시간↑
```

**메뉴 확장:**
```
기본: 위스키 니트(스트레이트)만 가능
해금1: 하이볼 (탄산수 제조기 구매 시)
해금2: 올드 패션드, 위스키 사워 (칵테일 장비 구매 시)
해금3: 프리미엄 칵테일 (전문 바 세트 + 레시피 습득 시)
해금4: 위스키 페어링 (안주 메뉴 추가 — 치즈, 초콜릿, 훈제 등)
해금5: 테이스팅 코스 (위스키 3종 비교 시음 세트)
```

### 직원 시스템 (1막 중반~)

직원은 "플레이어의 손이 부족해질 때" 해금. 처음엔 혼자 다 해야 함.

**1막 직원:**
```
서빙 알바:     손님에게 자동 서빙 (품질 낮음, 추천 불가)
               Lv1→2→3: 서빙 속도↑, 실수 확률↓
바텐더:        칵테일 자동 제조 가능 (플레이어보다 느림)
               Lv1→2→3: 칵테일 품질↑, 속도↑
매니저:        바 운영 일부 자동화 (재고 자동 주문 등)
               Lv1→2→3: 자동화 범위↑
```

**2막 직원 (추가):**
```
농부:          보리밭 자동 관리 (물주기, 수확)
               Lv1→2→3: 수확량↑, 품질↑
증류 보조:     매싱/발효 단계 자동화
               Lv1→2→3: 자동화 범위↑, 효율↑
배럴 관리사:   오크통 상태 모니터링, 문제 알림
               Lv1→2→3: 숙성 품질 보너스↑
```

직원 메카닉:
- 급여 (매일 고정 비용 — 수입보다 급여가 높으면 적자)
- 레벨업 (일하면서 경험치 → 능력 향상)
- 고용/해고 (시장에서 채용 가능, 능력치 랜덤)
- 직원이 있으면 플레이어는 다른 활동에 집중 가능 (증류소 일하면서 바는 직원이 운영)

### 증류소 업그레이드 트리 (2막~)

**시설:**
```
Pot Still:     소형 1기 → 중형 → 대형 → 2기 → 3기    — 일일 증류량↑
발효조:        나무통 1개 → 2개 → 스테인리스 → 대형    — 발효 배치↑, 품질↑
매싱 설비:     수동 → 반자동 → 자동                    — 매싱 효율↑, 시간↓
건조 시설:     없음(이탄 불가) → 소형 킬른 → 대형 킬른  — 몰팅 자체 처리 가능
숙성 창고:     오크통 5개 → 15 → 30 → 50 → 100       — 동시 숙성 수↑
보틀링 라인:   수동 → 반자동 → 자동                    — 병입 속도↑, 라벨 커스텀
```

**오크통 종류 (구매/해금):**
```
기본: 아메리칸 오크 (버번 캐스크) — 바닐라, 캐러멜
해금1: 유러피안 오크 (셰리 캐스크) — 건과일, 스파이시
해금2: 미즈나라 (일본 오크) — 백단향, 독특한 향
해금3: 와인 캐스크 — 베리, 탄닌
해금4: 포트 캐스크 — 달콤, 풍부
해금5: 한국 참나무 (게임 오리지널) — 고유 풍미, 최고급
```

### 플레이어 스킬 트리

플레이어 자신도 성장. 경험을 쌓으면 새로운 능력 해금:

**바텐더 스킬:**
```
위스키 감별안 Lv1~5:  손님 취향 힌트가 더 구체적으로 보임
서빙 속도 Lv1~3:     서빙 미니게임 시간 여유↑
칵테일 마스터 Lv1~3: 칵테일 레시피 해금, 품질 보너스
접객 스킬 Lv1~3:     팁 확률↑, 단골 전환 확률↑
```

**디스틸러 스킬 (2막~):**
```
증류 감각 Lv1~5:     컷 포인트 미니게임에서 가이드라인 표시
블렌딩 센스 Lv1~3:   블렌딩 시 풍미 예측 정확도↑
숙성 안목 Lv1~3:     오크통 상태/최적 숙성 시점 예측 가능
원재료 감별 Lv1~3:   보리/이탄 품질이 미리 보임
```

스킬 포인트: 매일 영업/작업 시 경험치 획득 → 레벨업 시 포인트 배분

### 성장 비용 밸런스 (대략적 가이드)

```
초기 자금 (퇴직금): 5,000,000원

[1막 바 업그레이드]
```

### 시각적 성장 (매우 중요!)

성장이 눈에 보여야 보람을 느낌:
- 바: 업그레이드마다 타일맵 변경 (허름한 벽→깔끔한 벽, 조명 추가, 가구 변화)
- 증류소: Pot Still 추가되면 실제로 맵에 등장
- 오크통 창고: 통이 하나씩 늘어남 (시각적으로 쌓이는 재산)
- 보리밭: 작은 밭 → 큰 밭으로 확장
- 직원: 바/증류소에 NPC가 돌아다니며 일하는 모습

## 아키텍처 패턴

### Autoload (싱글톤)
- `GameManager` — 날짜, 시간대, 현재 막(act 1/2/3), 게임 상태
- `SignalBus` — 전역 시그널 허브
- `Inventory` — 위스키 재고(사입+자체), 재료, 돈
- `Economy` — 가격, 거래, 시장 변동
- `StoryManager` — 스토리 진행 추적, 이벤트 트리거, 막 전환

### Signal 기반 통신
```gdscript
# SignalBus.gd
signal day_changed(day: int)
signal time_phase_changed(phase: String)  # "morning", "afternoon", "evening", "night"
signal money_changed(amount: int)
signal item_added(item_id: String, quantity: int)
signal whisky_completed(whisky: WhiskyData)
signal customer_served(satisfaction: float)
signal reputation_changed(new_value: float)
signal act_changed(act: int)              # 막 전환
signal story_event(event_id: String)      # 스토리 이벤트 발생
signal upgrade_purchased(upgrade_id: String)  # 업그레이드 구매
signal employee_hired(employee: EmployeeData) # 직원 고용
signal player_level_up(new_level: int)        # 플레이어 레벨업
signal skill_unlocked(skill_id: String)       # 스킬 해금
```

### State Machine
NPC, 게임 상태, 제조 공정 등:
```gdscript
enum State { IDLE, WALKING, INTERACTING, WORKING }
var current_state: State = State.IDLE

func _physics_process(delta):
    match current_state:
        State.IDLE: _idle_state(delta)
        State.WALKING: _walking_state(delta)
```

### Resource 기반 데이터
```gdscript
# whisky.gd
class_name WhiskyData
extends Resource

@export var name: String
@export var origin: String               # 산지 (1막: 구매처, 2막~: 자체)
@export var age_months: int
@export var flavor_profile: Dictionary   # {"smoky": 30, "sweet": 50, ...}
@export var quality: float               # 0.0 ~ 1.0
@export var base_price: int
@export var is_homebrew: bool = false     # 자체 제조 여부
```

### StoryManager 구조
```gdscript
# story_manager.gd
extends Node

var current_act: int = 1
var story_flags: Dictionary = {}
var current_day: int = 1

func check_story_triggers():
    match current_act:
        1: _check_act1_triggers()
        2: _check_act2_triggers()
        3: _check_act3_triggers()

func _check_act1_triggers():
    if current_day >= 5 and not story_flags.get("first_regular", false):
        trigger_event("first_regular")
    if current_day >= 25 and story_flags.get("whisky_shortage_count", 0) >= 3:
        trigger_event("act1_climax")
```

## 코딩 컨벤션

- 변수/함수: `snake_case`
- 클래스: `PascalCase`
- 상수: `UPPER_SNAKE_CASE`
- 시그널: `snake_case` (ex: `day_changed`, `act_changed`)
- 파일명: `snake_case.gd`, `snake_case.tscn`
- 주석: 한국어 OK
- 타입 힌트: 항상 사용 (`var speed: float = 80.0`)
- `@export`: 에디터 조정용
- `@onready`: _ready() 노드 참조

## 주의사항

- .tscn/.tres는 Godot 직렬화 포맷. GDScript와 다름
- .tscn에서 preload() 불가 → ExtResource("id") 사용
- TileMap은 에디터에서 편집이 효율적
- Autoload 등록은 Project Settings > Autoload에서 수동
- 1막 메카닉이 완성되어야 2막으로 진행. 1막이 재미없으면 2막을 붙여도 재미없음
- 기존 Phase 1~2 코드(증류, 바)는 2막에서 재활용. 삭제하지 말고 비활성화
